**Claimo** is a voucher / code redemption plugin for **PaperMC 1.21.1 – 26.2**.
Players redeem codes with a configurable command; each code runs one or more server
commands and can gate redemption behind **requirements** (blocks mined, playtime, …).

The requirement system is a **public API**, so addons can plug in new checks — e.g.
*"follows us on TikTok / YouTube / X / Instagram"*.

<Aside type="tip" title="Built for Paper">
	Claimo targets Paper and its API. It keeps `api-version: 1.21`, so it loads on every
	release from **1.21.1** upward, while optional features light up on newer versions.
</Aside>

## How it works

1. An admin defines a **voucher** — a code name, the command(s) it runs, and optional
   requirements and limits.
2. A player runs `/<command> <voucher>` (or clicks it in the GUI).
3. Claimo checks every requirement. If they all pass, the commands run and the player
   sees a success message; otherwise a checklist of met / unmet requirements is shown.

## Version compatibility

| Feature | Minimum version |
| --- | --- |
| Core plugin (redeem, GUI, requirements, storage) | Paper **1.21.1** |
| In-game creator — `/code create`, `edit`, `delete` | Paper **1.21.7+** <Badge text="Dialog API" variant="tip" /> |

<Aside type="note">
	On 1.21.1–1.21.6 the plugin still loads and everything else works — only the Dialog
	creator is disabled. See [The in-game creator](/guides/creator/) for details.
</Aside>
