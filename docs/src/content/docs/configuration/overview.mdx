Claimo's configuration is split across a few files in the plugin's data folder, each
created on first run. Edit them, then run `/<command> reload`.

<FileTree>
- plugins/Claimo/
  - config.yml core settings — command, sound, logging, storage
  - messages.yml all player-facing text (MiniMessage)
  - gui.yml the voucher list GUI
  - vouchers/ one .yml per code — the file name is the code
    - test.yml
</FileTree>

## Applying changes

```text
/code reload
```

reloads config and voucher files without a restart.

<Aside type="caution" title="Restart-only keys">
	A few settings are read once at startup and need a **server restart**, not a reload:
	- `command:` — the root command name.
	- `storage.type:` — the storage backend.
</Aside>
